# Результаты замеров

Проект: `sessions-s07`

## Сравнение REST и gRPC

В рамках лабораторной работы были выполнены замеры времени для 1000 запросов к REST API и 1000 запросов к gRPC Unary методу.

### Условия замеров

- запуск на одной машине;
- адрес: `localhost`;
- REST-сервис запускался локально;
- gRPC-сервис запускался на порту `50051`;
- для сравнения использовался Unary метод gRPC.

### Пример результатов

- REST: `0.8500 sec`
- gRPC: `0.4200 sec`

Результаты на моей машине:  

- REST: `1.2889 sec`
- gRPC: `0.5779 sec`

> Точные значения могут отличаться на разных компьютерах и при разных условиях запуска.

## Вывод

gRPC показал более высокую производительность по сравнению с REST.

Причины:
- используется бинарный формат Protobuf вместо JSON;
- меньше размер передаваемых данных;
- сериализация и десериализация обычно быстрее;
- gRPC работает поверх HTTP/2.

При этом REST остается проще для отладки и удобнее для публичных API, а gRPC лучше подходит для внутренних сервисов и микросервисной архитектуры.
