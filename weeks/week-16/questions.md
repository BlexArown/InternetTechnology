### Что такое OWASP Top 10? Назовите 3 любые уязвимости оттуда.

OWASP Top 10 — это список самых критичных и распространённых уязвимостей веб-приложений, составляемый организацией OWASP (Open Web Application Security Project).

Он используется как стандарт для проведения аудита безопасности и разработки безопасных приложений.

Примеры уязвимостей из OWASP Top 10:
- Broken Access Control — доступ к чужим данным через изменение параметров (например, ID).
- Injection — SQL/Command Injection через пользовательский ввод.
- Security Misconfiguration — неправильные настройки (например, включённый debug или открытые порты).

### Чем Аутентификация отличается от Авторизации?

Аутентификация (Authentication) — это процесс проверки личности пользователя.
Ответ на вопрос: "Кто ты?"

Пример: ввод логина и пароля, проверка JWT токена.

Авторизация (Authorization) — это процесс проверки прав доступа.
Ответ на вопрос: "Что тебе можно?"

Пример: может ли пользователь удалить отзыв или видеть чужие данные.

Итого:
- Аутентификация -> кто пользователь
- Авторизация -> что ему разрешено

### Что такое SQL Injection и как ORM помогают от неё защититься?

SQL Injection — это атака, при которой злоумышленник вставляет вредоносный SQL-код в пользовательский ввод, чтобы изменить поведение запроса.

Пример:
' OR '1'='1

Если запрос формируется как строка:
SELECT * FROM users WHERE name = 'input'

то злоумышленник может получить доступ ко всем данным.

ORM (Object-Relational Mapping) защищает за счёт:
- параметризованных запросов;
- автоматического экранирования данных;
- запрета прямой вставки SQL-кода.

Это снижает риск SQL Injection.

### Почему нельзя хранить секреты (API ключи, пароли БД) в коде репозитория?

Потому что репозиторий может быть:
- публичным;
- доступным другим разработчикам;
- с историей коммитов (секрет остаётся даже после удаления).

Если секрет попадёт в репозиторий:
- злоумышленник получит доступ к базе данных;
- можно скомпрометировать внешние сервисы;
- возможны финансовые потери.

Правильный подход:
- использовать переменные окружения;
- использовать Secret Manager (Kubernetes Secrets, GitHub Secrets);
- не коммитить .env файлы.

### Что такое "Man-in-the-Middle" (MitM) атака и как HTTPS защищает от неё?

MitM (Man-in-the-Middle) — это атака, при которой злоумышленник перехватывает трафик между клиентом и сервером.

Он может:
- читать данные;
- изменять запросы;
- подменять ответы.

HTTPS защищает за счёт:
- шифрования (данные нельзя прочитать);
- сертификатов (проверка подлинности сервера);
- TLS (защита от подмены данных).

Таким образом злоумышленник не может перехватить или изменить трафик.

### Зачем нужен принцип "минимальных привилегий" (Least Privilege)?

Принцип минимальных привилегий означает, что пользователь или сервис получает только те права, которые необходимы для работы, и не больше.

Зачем это нужно:
- уменьшает риск утечки данных;
- ограничивает последствия взлома;
- снижает поверхность атаки.

Пример:
- сервису не нужен доступ ко всей базе — только к своей таблице;
- пользователь не должен иметь права администратора.

Это один из ключевых принципов безопасности систем.
