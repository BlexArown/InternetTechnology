1. В чем разница между Forward Proxy и Reverse Proxy?  

Forward Proxy (прямой прокси) — работает от имени клиента. Он скрывает клиента от сервера.  
Схема: Клиент → Forward Proxy → Интернет → Сервер;  

Reverse Proxy (обратный прокси) — работает от имени сервера. Он скрывает реальные серверы от клиента.  
Схема: Клиент → Reverse Proxy (Nginx) → Backend сервис;  

2. Что такое Upstream в терминологии Nginx?  

Upstream — это группа backend-серверов, к которым Nginx пересылает запросы.  

	upstream photos-service {  
 	   server photos-svc-s07:8255;  
	}  

Теперь Nginx может направлять запросы в этот upstream. То есть upstream = backend сервис.  

3. Зачем скрывать реальные IP-адреса сервисов от клиента?  

Причины:  
	1. Безопасность  
		Клиент не может напрямую атаковать backend;  
	2. Централизация доступа  
		Все запросы идут через одну точку (Gateway);  
	3. Гибкость  
		Можно менять backend, не меняя клиент;  
	4. Защита инфраструктуры  
		IP и архитектура не раскрываются;  

4. Какие еще задачи может выполнять API Gateway (кроме маршрутизации)?  

	1. Аутентификация и авторизация  
		Проверка токенов, пользователей;  
                           2. Балансировка нагрузки  
	                           Распределяет запросы между несколькими серверами;  
	                           Client → Gateway → Server1 → Server2;  
                           3. Логирование  
	                            Запись всех запросов.  
	4. Кэширование  
		Ускоряет ответы;  
	5. Rate limiting  
		Ограничение количества запросов;  

5. Что произойдет, если Gateway упадет? Как защититься?  

Если Gateway упадет:  
	Клиенты не смогут получить доступ к сервисам, вся система станет недоступной;  

Потому что Gateway — единая точка входа.  

Как защититься:  
	1. Запустить несколько Gateway  
		Client → Load Balancer → Gateway1 → Gateway2;  
	2. Использовать балансировщик нагрузки  
		Например: Nginx, HAProxy, Kubernetes Service;  
	3. Использовать отказоустойчивость (Failover)  
		Автоматическое переключение на резервный Gateway;
