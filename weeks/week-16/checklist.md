# Security Checklist для reviews-s07

## OWASP Top 10 проверки

- [x] Проверка на SQL Injection.
- [x] Проверка на Command Injection.
- [x] Проверка валидации входных данных.
- [x] Проверка поля rating: значение должно быть числом в допустимом диапазоне.
- [x] Проверка Broken Access Control: нельзя получать чужие данные по изменению ID.
- [x] Проверка Authentication: защищенные методы должны требовать авторизацию.
- [x] Проверка JWT/token validation: токен должен проверяться на сервере.
- [x] Проверка Sensitive Data Exposure: секреты не должны храниться в коде.
- [x] Проверка Cryptographic Failures: пароли и ключи не должны храниться открыто.
- [x] Проверка Security Misconfiguration: debug-режим должен быть выключен.
- [x] Проверка CORS: нельзя разрешать все origin в production.
- [x] Проверка Rate Limiting: API должен ограничивать частоту запросов.
- [x] Проверка Logging & Monitoring: ошибки и подозрительные запросы должны логироваться.
- [x] Проверка устаревших зависимостей.
- [x] Проверка контейнера: не запускать приложение от root.
